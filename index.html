Project Goals

Advancing multimodal sensing, data curation, and AI-enabled domain awareness

This project builds on the PANDAWN network to collect, curate, and analyze real-world multimodal sensing data from geographically distributed sites. These efforts support data-driven algorithm development, automated labeling, and AI-enabled analysis for nonproliferation-relevant applications.

Acquire, label, and curate PANDAWN multi-sensor data

The project will operate the PANDAWN network for sustained multimodal data collection and curate datasets for dissemination to the NA-22 community. Machine learning and data science methods will be integrated into the acquisition and curation process to support automated labeling and improve the utility of collected data products.

Explore multi-sensor data and radiological pattern-of-life

The project will analyze radiological pattern-of-life across time, location, environmental conditions, and deployment contexts. These analyses will support the development of new algorithms for multi-sensor data interpretation and, where appropriate, incorporate validated methods into the labeling and curation pipeline.

Evaluate AI methods for multimodal analysis and domain awareness

The project will evaluate AI methods for fusing heterogeneous sensing modalities and extracting actionable information from realistic, geographically distributed data. These efforts will help advance domain-awareness capabilities for complex real-world environments relevant to nonproliferation applications.

PANDAWN Network

A remotely reconfigurable Rad/Nuc sensing network designed to evolve over time

PANDAWN is an urban sensing network composed of nominally 14 strategically placed sensor nodes across Chicago. The network combines radiation/nuclear sensing, environmental sensing, edge computing, cloud-connected analysis, and remote reconfiguration to support long-term multimodal data collection and algorithm development in realistic urban environments.

Distributed urban sensing

Strategically placed nodes provide geographically diverse measurements across Chicago. The network supports data collection across varied urban environments and deployment contexts, enabling analysis of spatial patterns and site-specific sensing conditions.

Built-in edge computing

PANDAWN supports AI/ML and conventional computing close to the data source. This allows sensor data to be processed locally, enabling responsive analysis while reducing dependence on centralized processing for every workflow.

Software and hardware extensibility

The network is designed to be updated over time to support new sensors, workflows, and algorithms. This extensibility allows PANDAWN to evolve with emerging mission needs while operating within the practical limits of deployed hardware and software.

Closed-loop edge-cloud operation

PANDAWN enables jobs to be scheduled and coordinated across edge nodes and cloud resources. This closed-loop architecture supports feedback between cloud-level analysis and edge-level execution, allowing workflows to adapt as new data and context become available.

Context-aware execution

Edge nodes can support adaptive job execution based on local observations, system state, and broader network context. This enables more intelligent use of sensing, compute, and communication resources across the network.

Network-wide aggregation and analysis

Data from multiple nodes can be aggregated for cloud-based analysis, visualization, and algorithm evaluation. These network-wide views support spatial, temporal, and multimodal analysis across deployment sites.

PANDAWN Network Data

Data streams and curated products for analysis, labeling, and algorithm development

PANDAWN provides access to multiple data products, ranging from near-real-time data streams to QA/QC-processed, labeled, and long-term curated datasets. Access is managed through approval and authentication workflows to support responsible use by the research and NA-22 communities.

Available data types

  • Contextual information: Weather measurements including temperature, pressure, humidity, and rainfall; activity and other contextual signals inferred from images; and LiDAR point-cloud data.
  • Radiation/Nuclear data: List-mode Digibase data collected from PANDAWN Rad/Nuc sensing systems.
  • Raw multimodal data: Two-minute video captures collected together with the other synchronized data streams.

Data Endpoints

Real-time data streams

Near-real-time PANDAWN data are available with approximately 30-second latency through the PANDAWN Data Portal. Time-series data can be accessed through a Python API using JSON, while larger binary objects are available through HTTPS endpoints.

QA/QC and labeled data

Quality-controlled and labeled data products are currently available with approximately one-day latency through the Berkeley Data Cloud. These products support downstream analysis, algorithm testing, and model development using curated PANDAWN observations.

Long-term curated datasets

Long-term curated datasets are available through the Berkeley Data Cloud and are planned for release through Alexandria with DOI support. These datasets provide durable, citable data products for research, evaluation, and community use.

Data Roadmap

Lower-latency QA/QC and labeled data

Future processing will reduce QA/QC and labeled-data latency from approximately one day to a matter of hours. This will be enabled by moving processing into the PANDAWN Data Cloud at Argonne, where data can be ingested directly from nodes in real time.

Expanded curated data access

Long-term curated datasets will continue to be available through the Berkeley Data Cloud and will also be made available through Alexandria. This dual-access model will support both operational data use and long-term archival dissemination.

PANDAWN Platform

An AI@Edge system for RadFusion and distributed intelligence

The PANDAWN platform builds on the Waggle AI@Edge ecosystem to support distributed sensing, edge computing, and multimodal data fusion for radiation/nuclear domain awareness. Its modular architecture allows heterogeneous sensors, GPUs, accelerators, and containerized AI workloads to operate together across deployed edge nodes and cloud resources.

Modular edge sensing nodes

PANDAWN nodes combine heterogeneous sensors with edge-local compute resources, including GPUs and accelerators. This provides a flexible platform for collecting, preprocessing, and analyzing multimodal data close to the source.

Containerized AI/ML workloads

Machine learning and conventional analysis workflows can be packaged as containerized workloads and deployed directly to edge nodes. This enables rapid updates, reproducible execution, and experimentation with new algorithms in the field.

Cloud and edge orchestration

PANDAWN supports coordinated execution across edge and cloud resources. This allows workloads to run where they are most effective, whether on-node for low-latency analysis or in the cloud for network-wide fusion and aggregation.

Proven deployment foundation

The underlying Waggle AI@Edge platform has been demonstrated in real urban, environmental, and scientific sensing deployments. PANDAWN extends this foundation toward RadFusion and radiation/nuclear sensing applications.

End-to-end data fusion pipeline

PANDAWN supports the full data pipeline from sensor acquisition and preprocessing to on-node inference, feature extraction, metadata generation, local fusion, and cloud-scale analysis. This pipeline enables networked detection across distributed sensing nodes.

On-node inference and feature generation

Edge nodes can support inference tasks such as object detection, source attribution, and anomaly detection. These capabilities allow each node to generate features and metadata that can be used for local, regional, and global fusion.

Local and networked detection

PANDAWN enables fusion at multiple scales, from local node-level analysis to regional and network-wide cloud fusion. This supports richer domain awareness by combining observations across sensors, locations, and time.
