PANDAWN Network Data

Data streams and curated products for analysis, labeling, and algorithm development

PANDAWN provides access to multiple data products, ranging from near-real-time data streams to QA/QC-processed, labeled, and long-term curated datasets. Access is managed through approval and authentication workflows to support responsible use by the research and NA-22 communities.

Available data types

  • Contextual information: Weather measurements including temperature, pressure, humidity, and rainfall; activity and other contextual signals inferred from images; and LiDAR point-cloud data.
  • Radiation/Nuclear data: List-mode Digibase data collected from PANDAWN Rad/Nuc sensing systems.
  • Raw multimodal data: Two-minute video captures collected together with the other synchronized data streams.

Raw data streams and acquisition characteristics

  • Meteorological streams: Rain gauge measurements are collected every second, while temperature, pressure, and humidity are collected every 30 seconds.
  • Radiation/Nuclear streams: PANDAWN collects list-mode radiation data together with online calibration data.
  • Continuous low-rate acquisition: Continuous meteorological and radiation data collection produces approximately 2.5 GB per node per day.
  • Camera and LiDAR streams: Camera data are collected at approximately 20–30 Hz, and LiDAR data are collected at approximately 10 Hz.
  • Selected multimodal acquisition: Selected acquisition windows, such as six two-minute captures per day, combine radiation, camera, and LiDAR data and produce approximately 12 GB per node per day.
  • Time-synchronized multimodal acquisition: ROS-based acquisition applications synchronize data streams across modalities, enabling aligned radiation, meteorological, camera, and LiDAR observations for multimodal analysis and RadFusion workflows.
  • QA/QC and real-time access: QA/QC-processed data products are produced for curated analysis, while non-QC’ed data are available in real time through the Sage portal and API.

Data Endpoints

Real-time data streams

Near-real-time PANDAWN data are available with approximately 30-second latency through the PANDAWN Data Portal. Time-series data can be accessed through a Python API using JSON, while larger binary objects are available through HTTPS endpoints.

QA/QC and labeled data

Quality-controlled and labeled data products are currently available with approximately one-day latency through the Berkeley Data Cloud. These products support downstream analysis, algorithm testing, and model development using curated PANDAWN observations.

Long-term curated datasets

Long-term curated datasets are available through the Berkeley Data Cloud and are planned for release through Alexandria with DOI support. These datasets provide durable, citable data products for research, evaluation, and community use.

Data Roadmap

Lower-latency QA/QC and labeled data

Future processing will reduce QA/QC and labeled-data latency from approximately one day to a matter of hours. This will be enabled by moving processing into the PANDAWN Data Cloud at Argonne, where data can be ingested directly from nodes in real time.

Expanded curated data access

Long-term curated datasets will continue to be available through the Berkeley Data Cloud and will also be made available through Alexandria. This dual-access model will support both operational data use and long-term archival dissemination.

Data Labels

Automated labeling for radiological and contextual data products

PANDAWN supports automated labeling of radiological and contextual camera data to enable downstream analysis, AI/ML model development, and curated data product generation. Labeled data products are produced at approximately 6.25 GB per node per day and are made available through the Berkeley Data Cloud with approximately one-day latency.

Radiological labels

Radiological labels are generated using LBNL’s Background Adaptive Radiation Detection package, or BARD. BARD supports in-situ, ab-initio background model learning, training, and updating, along with NMF-based background anomaly detection and isotope identification.

Radiological label products

Radiological label products include bin-mode data with 0.5-second integration time, timestamps, count rates, livetimes, calibration information, meteorological context, background anomaly detection metrics, and isotope identification metrics.

Camera-based contextual labels

Contextual camera labels are generated using YOLOv10l for object detection and Tryolabs’ Norfair for object tracking. These methods provide frame-level contextual information to support multimodal analysis with radiological observations.

Camera label products

Camera label products include frame-by-frame bounding boxes, object classes, confidence levels, and frame-to-frame track identifiers. These labels help characterize activity and context near each sensing location.

Monthly curated collections

Data labels for each node are curated into monthly collections on the Berkeley Data Cloud, providing organized label products for analysis, evaluation, and model development.
